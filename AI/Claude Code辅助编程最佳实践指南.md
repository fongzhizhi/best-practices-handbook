> **CreateDate**: *2026-03-30*
>
> **UpdateDate**: *2026-03-30*

> 基于 Claude Code 的 Commands、Skills 和 Agent 构建科学工作流。

---

## 一、三大核心能力

| 能力         | 一句话定义                                             | 典型场景                                 |
| ------------ | ------------------------------------------------------ | ---------------------------------------- |
| **Commands** | 以 `/` 开头的快捷指令，执行预定义操作或启动工作流      | 快速生成测试、创建文件、启动规范流程     |
| **Skills**   | 预置的“专业能力包”，加载后改变 AI 的行为模式与知识范围 | 统一代码风格、领域专家模式、遵循团队规范 |
| **Agent**    | 自主规划、调用工具、执行多步任务的智能体               | 跨文件重构、多仓库分析、批量操作         |

**核心理念**：  

- **Commands** 适合“一键触发”的固定动作  
- **Skills** 负责“如何思考”与“遵循什么标准”  
- **Agent** 负责“如何行动”与“完成复杂目标”

三者结合，可以从“随性对话”升级为“工程化 AI 协作”。

## 二、场景化最佳实践

### 场景 1：写需求 / 设计文档（新功能 / 新项目）

**推荐组合**：**Commands (Spec 工作流) + Agent**

**最佳实践**：
1. **初始化 Spec 工作流**。
2. **创建 Spec**：
   - 使用 `/spec-create 功能名称 "简要描述"` 命令。
   - 系统自动创建需求、设计、任务三个文档的骨架。
3. **编写需求与设计**：
   - 使用 `/spec-requirements` 填写详细需求（可让 AI 辅助完善）。
   - 使用 `/spec-design` 输出技术设计，可结合 **Skill**（如 `architecture-design`）确保设计符合团队规范。
4. **拆解任务**：使用 `/spec-tasks` 生成任务清单，每个任务粒度适中（1-2 小时工作量）。
5. **执行**：使用 `/spec-execute` 依次执行任务，内部由 Agent 自动完成代码编写、测试生成、文档更新。

**示例提示词**（在需求阶段）：
```
使用 architecture-design skill 帮我完善这个功能的系统设计，考虑高可用和扩展性。
```

**为什么高效**：
- 规范驱动，避免方向跑偏。
- 文档与代码同步演进，可追溯。
- Agent 自动执行，减少人工干预。

---

### 场景 2：注释补全（为现有代码添加 JSDoc / 注释）

**推荐组合**：**Command (自定义) + Skill (团队注释规范)**

**最佳实践**：
1. **创建自定义 Command**（如 `/add-jsdoc`）：
   - 在 `.claude/commands/add-jsdoc.md` 中编写指令，内容例如：
     ```markdown
     请为当前选中的函数或类添加 JSDoc 注释，遵循团队规范：
     - 描述功能、参数类型和含义、返回值
     - 标注 @example（如有必要）
     - 使用 @param {type} name - description 格式
     ```
2. **创建团队 Skill**（如 `team-jsdoc`）：
   - 在 `.claude/skills/team-jsdoc.md` 中定义详细的 JSDoc 规范、示例和常见错误。
3. **使用**：
   - 选中需要添加注释的代码块。
   - 输入 `/add-jsdoc` 或直接说“使用 team-jsdoc skill 为这段代码添加注释”。
   - AI 会按照规范生成注释，并可自动插入。

**示例提示词**：
```
使用 team-jsdoc skill 为这个函数生成注释，同时检查是否有遗漏的参数说明。
```

**为什么高效**：
- 规范统一，避免五花八门的注释风格。
- 一键生成，节省手工编写时间。
- 可批量处理（Agent 可对多个文件执行）。

### 场景 3：写单元测试

**推荐组合**：**Command + Agent（批量生成）**

**最佳实践**：
1. **使用内置 `/test` 命令**：选中一个函数或类，输入 `/test`，AI 会立即生成对应的测试用例（适合单文件）。
2. **批量生成**：对于多个文件或模块，可以启动 Agent 模式：
   ```
   /agent 为 src/services/ 目录下所有 .ts 文件生成单元测试，使用 Jest 框架，覆盖正常流程和边界条件。测试文件放在 tests/unit/services/ 下。
   ```
3. **增强测试质量**：可以结合 **Skill**（如 `test-expert`）定义团队测试规范（如测试命名、mock 策略、覆盖率要求）。

**示例提示词**（使用 Skill 强化）：
```
使用 test-expert skill 为这个支付模块生成测试，确保覆盖所有异常分支，并 mock 外部 API。
```

**为什么高效**：
- 快速生成基础测试，减少重复劳动。
- 结合 Skill 保证测试质量，避免“为了测试而测试”。
- Agent 可并行处理多个文件，适合遗留系统补测试。

### 场景 4：修 Bug

**推荐组合**：**Agent + Skill（调试专家）**

**最佳实践**：
1. **定位问题**：将错误堆栈、复现步骤提供给 AI，让它分析可能的原因。
   ```
   我遇到了这个错误：TypeError: Cannot read property 'id' of undefined in orderService.js:45。请分析可能的原因，并定位到具体代码行。
   ```
2. **加载调试 Skill**：使用 `debugging` skill 让 AI 采用调试专家思维（系统化排查、考虑边界条件）。
3. **自动修复**：启动 Agent 让其自主修复，同时要求生成回归测试：
   ```
   /agent 修复这个 bug，并添加一个会失败的测试用例来验证修复。修复后运行所有相关测试确保无回归。
   ```
4. **审查与提交**：Agent 执行过程中会在关键点（如修改核心逻辑）请求确认，人工审查后继续。

**示例提示词**：
```
使用 debugging skill 分析这个 bug，然后帮我修复，并补充单元测试避免再次出现。
```

**为什么高效**：
- AI 快速定位根源，避免无头苍蝇式排查。
- 自动添加测试，防止二次引入。
- Agent 可同时更新相关文档（如错误处理说明）。

### 场景 5：根据代码梳理文档

**推荐组合**：**Agent + Skill（文档专家）**

**最佳实践**：
1. **明确文档类型**：告诉 AI 要生成什么文档（README、API 文档、架构设计、维护手册等）。
2. **加载文档 Skill**：例如 `skill: technical-writer`，定义文档风格、结构模板。
3. **启动 Agent 扫描**：
   ```
   /agent 分析 src/core/ 目录，生成一份架构设计文档，包含模块职责、关键流程、类图（Mermaid）、配置说明。使用 technical-writer skill 确保文档结构清晰。
   ```
4. **迭代优化**：Agent 输出初稿后，可以人工补充业务背景，再让 AI 润色。

**示例提示词**：
```
使用 technical-writer skill 为这个支付模块生成维护手册，重点说明退款流程、常见错误码和排查步骤。
```

**为什么高效**：
- 从代码反向生成文档，解决遗留系统无文档的痛点。
- Skill 保证文档风格统一，符合团队标准。
- Agent 可跨文件分析，生成更全面的视图。

---

### 场景 6：重构代码

**推荐组合**：**Spec 工作流（Commands + Agent） + Skill（重构模式）**

**最佳实践**：
1. **先分析后动手**：使用 Agent 分析目标模块，输出重构建议。
   ```
   /agent 分析 src/payment/ 模块，识别代码坏味道，输出重构方案，保持对外接口不变。
   ```
2. **创建 Spec**：对于较大重构，使用 `/spec-create` 生成需求/设计文档，明确重构步骤和验收标准。
3. **加载重构 Skill**：如 `refactoring-patterns`，定义团队常用的重构手法（提取函数、引入接口等）。
4. **分步执行**：使用 `/spec-execute` 逐步实施重构，每一步完成后运行测试验证。
   - Agent 在修改代码后会自动运行测试，确保功能不变。
5. **更新文档**：重构完成后，使用 Agent 更新相关文档。

**示例提示词**（重构阶段）：
```
使用 refactoring-patterns skill 重构这个类，将重复代码提取到公共方法，并保持所有公共方法签名不变。
```

**为什么高效**：
- 分步执行降低风险。
- Skill 确保重构手法规范。
- 测试自动验证，避免破坏现有功能。

## 三、综合工作流建议

在日常开发中，建议按以下流程融合使用这三种能力：

1. **新需求 / 大改动**  
   → 使用 **Spec 工作流（Commands）** 启动规范流程。  
   → 在需求/设计阶段，加载相关 **Skills** 确保输出质量。  
   → 执行阶段由 **Agent** 自动完成代码编写、测试、文档更新。

2. **日常小任务（测试、注释、小修小补）**  
   → 使用 **自定义 Commands** 快速完成。  
   → 可同时加载 **Skills** 确保风格统一。

3. **复杂探索性任务（跨模块分析、重构、文档梳理）**  
   → 启动 **Agent** 模式，描述目标。  
   → 必要时加载 **Skills** 限制行为范围（如“只修改 src/ 下的文件”）。

4. **团队协作**  
   → 将公共 **Commands** 和 **Skills** 提交到代码库（`.claude/` 目录），保证团队使用一致的 AI 辅助标准。  
   → 在 `CLAUDE.md` 中配置默认加载的 Skills，让 AI 自动遵循项目规范。

## 四、落地建议

- **从小处着手**：先为团队创建 1-2 个常用 Command（如 `/add-test`、`/add-jsdoc`），让大家尝到甜头。
- **逐步建立 Skills**：收集团队的代码规范、测试要求、文档模板，沉淀为 Skills。
- **利用 Agent 处理脏活累活**：把重复性高、逻辑清晰但繁琐的任务（如批量改包名、统一日志格式）交给 Agent。
- **定期复盘**：根据实际使用效果调整 Commands 和 Skills 的内容，让 AI 越来越懂你的项目。

通过科学地运用 **Commands、Skills 和 Agent**，你不仅能让 AI 更高效地协助编程，还能将团队的最佳实践固化下来，让整个开发过程更可控、更可维护。
