请帮助用户切换到指定分支，并确保该分支处于最新状态。本指令可独立使用，能智能处理常见障碍。

## 执行步骤

### 1. 获取目标分支名
- 若用户执行 `/git-checkout <分支名>`，则使用 `<分支名>`。
- 若用户仅输入 `/git-checkout`，则询问用户需要切换到哪个分支。

### 2. 场景预检：当前分支与目标分支相同
- 若目标分支名与当前分支名相同，提示用户：

  ```txt
  ℹ️ 已在分支 `<分支名>` 上。
  是否需要拉取最新代码？(y/n)
  ```

    - 若选 `y`，直接跳至步骤 6（拉取最新代码）。

    - 若选 `n`，流程结束。


### 3. 场景预检：未提交变更处理
- 执行 `git status --porcelain` 检查工作区和暂存区状态。

- 若存在未提交变更，提示用户：

  ```txt
  ⚠️ 当前分支有未提交的变更，切换分支可能导致内容丢失。
  请选择处理方式：
  A. 自动暂存变更（调用 /git-stash），切换后再恢复
  B. 仅暂存变更，切换后不自动恢复
  C. 取消切换，手动处理
  ```

    - 若选 `A`：调用 `/git-stash` 暂存变更，记录 stash 引用，切换后询问是否执行 `git stash pop`。

    - 若选 `B`：调用 `/git-stash` 暂存变更，切换后不恢复。

    - 若选 `C`：终止流程，提示用户手动处理。


### 4. 检查本地分支是否存在
- 执行 `git branch --list <分支名>` 检查本地分支。
- 若存在，直接切换：`git checkout <分支名>`。
- 若不存在，进入下一步。

### 5. 检查远程分支是否存在
- 执行 `git fetch origin` 更新远程分支列表。

- 执行 `git branch -r --list origin/<分支名>` 检查远程分支。

- 若远程存在：
  - 检查本地是否存在同名但无跟踪关系的分支（即步骤 4 为不存在，但 `git branch` 可能因大小写敏感等问题遗漏，可进一步确认）。
  - 基于远程分支创建本地跟踪分支：`git checkout -b <分支名> origin/<分支名>`。
  
- 若远程也不存在：

  提示用户并终止流程：

  ```txt
  ❌ 分支 `<分支名>` 在本地和远程均不存在。
  请确认分支名是否正确，或使用 `/git-branch` 创建新分支。
  ```

### 6. 确保上游跟踪与拉取最新代码
- 切换成功后，检查当前分支是否已关联远程跟踪分支：
  - 执行 `git rev-parse --abbrev-ref @{upstream}` 检查。
  
- 若未关联，但远程存在同名分支，则自动设置跟踪：

  ```bash
  git branch --set-upstream-to=origin/<分支名> <分支名>
  ```

- 执行 `git pull` 拉取最新代码。
  - 若拉取时发生冲突（如本地有未推送提交且与远程有分歧），提示用户：
  
    ```txt
    ⚠️ 拉取远程代码时发生冲突。
    请选择处理方式：
    A. 变基（git pull --rebase）
    B. 合并（git pull --no-rebase）
    C. 取消，稍后手动处理
    ```
  
  - 根据用户选择执行相应命令，若有冲突则给出解决指引。

### 7. 恢复暂存内容（若步骤 3 选择 A）
- 若用户在步骤 3 选择了自动暂存并恢复，此时询问：
  ```txt
  是否恢复之前暂存的变更？(y/n)
  ```
  - 若选 `y`，执行 `git stash pop`。若有冲突，提示手动解决。
  - 若选 `n`，提示用户可通过 `git stash list` 查看并手动恢复。

### 8. 输出结果
```
✅ 已切换到分支：<分支名>
✅ 已拉取最新代码。
```

## 使用示例

- `/git-checkout develop` — 切换到 develop 分支
- `/git-checkout main` — 切换到 main 分支
- `/git-checkout` — 交互式询问分支名
